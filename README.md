## Описание проекта

Данный проект реализует полнофункциональное веб-приложение на Java с использованием современного стека технологий Jakarta EE. Приложение позволяет пользователю проверять попадание точки в заданную геометрическую область на координатной плоскости, визуализировать результаты на интерактивном графике и сохранять историю всех проверок в базе данных. Реализована система кэширования с возможностью динамического переключения между различными типами кэшей для оптимизации производительности.

---

## Основные функции

1. **Проверка попадания точки в область**: На основе введённых координат X, Y и радиуса R система определяет, находится ли точка в заданной геометрической области
2. **Валидация входных данных**: Строгая проверка корректности введённых значений с поддержкой пользовательских сообщений об ошибках
3. **Визуализация на интерактивном графике**: Отображение точек и область наCanvas с возможностью клика для выбора координат
4. **История проверок**: Хранение всех результатов в базе данных с отображением таблицы результатов
5. **Кэширование результатов**: Поддержка двух типов кэшей (Caffeine и Redis) с возможностью переключения на лету
6. **Отслеживание производительности**: Запись времени выполнения каждой проверки в наносекундах

### Слои приложения:

1. **Presentation Layer (Презентационный слой)**
   - JSF Managed Beans (PointCheckBean, ResultsBean, CacheSwitcherBean)
   - XHTML шаблоны с PrimeFaces компонентами
   - Клиентская валидация и интерактивный Canvas

2. **Business Logic Layer (Слой бизнес-логики)**
   - AreaCheckService: основная логика проверки попадания точки в область
   - CachingAreaCheckService: оборачивание сервиса с кэшированием
   - CacheProvider и его реализации: работа с различными кэшами

3. **Data Access Layer (Слой доступа к данным)**
   - Repository: выполнение операций CRUD над данными
   - JPA Entity: описание структуры таблицы базы данных

## Ключевые компоненты

### 1. Контроллеры (Controllers)

**PointCheckBean** - управляет вводом координат X, Y и радиуса R
- Взаимодействует с CachingAreaCheckService для проверки точки
- Вызывает ResultsBean для обновления списка результатов
- Поддерживает двусторонее связывание данных (two-way binding)

**ResultsBean** - управление историей результатов
- Получает список всех результатов из базы данных
- Отображает таблицу с результатами в реальном времени

**CacheSwitcherBean** - переключение типа кэша
- Позволяет пользователю выбирать между Caffeine и Redis
- Обновляет активный провайдер кэша во время работы

### 2. Сервисы (Services)

**AreaCheckService** - основной сервис проверки
```
Задачи:
- Математическая проверка, находится ли точка в области
- Сохранение результата в базу данных
- Измерение времени выполнения операции
```

**CachingAreaCheckService** - декоратор с кэшированием
```
Задачи:
- Проверка кэша перед выполнением проверки
- Сохранение результатов в кэш после выполнения
- Динамическое переключение между Caffeine и Redis
- Управление жизненным циклом кэша
```

### 3. Валидаторы (Validators)

Реализованы пользовательские валидаторы JSF:

- **XValidator**: X должно быть в диапазоне [-5; 5]
- **YValidator**: Y должно быть в диапазоне [-3; 3]
- **RValidator**: R должно быть в диапазоне [1; 5]

Все валидаторы выполняют проверку как на сервере, так и предусмотрены клиентские проверки через JavaScript.

### 4. Кэширование

Реализована система с двумя провайдерами кэша:

**CaffeineCacheProvider** (по умолчанию)
- Кэширование в памяти приложения
- Быстрый доступ
- Данные теряются при перезагрузке приложения

**RedisCacheProvider** (опционально)
- Распределённое кэширование
- Персистентность данных
- Возможность совместного использования кэша несколькими экземплярами приложения

### 5. Сущности JPA

**ResultEntity** - представляет запись результата в БД
```java
Поля:
- id: Long - первичный ключ
- userId: double - ID пользователя
- x: double - координата X
- y: double - координата Y
- r: double - радиус
- isHit: boolean - результат проверки
- timestamp: Date - время проверки
- executionTimeNano: long - время выполнения в наносекундах
```

### 6. DTO (Data Transfer Objects)

- **PointDTO**: передача координат и радиуса между слоями
- **ResultDTO**: передача результатов проверки с метаданными

## Технические детали реализации

### Управление жизненным циклом

- **@ApplicationScoped** - сервисы и кэши существуют на уровне приложения
- **@ViewScoped** - контроллеры форм существуют на уровне представления
- **@PostConstruct** - инициализация компонентов после создания
- **@Inject** - внедрение зависимостей через CDI

### Логирование

Используется встроенный `java.util.logging.Logger` для отслеживания:
- Операций кэширования
- Сохранения результатов в БД
- Переключения типа кэша

### Конфигурация базы данных

**persistence.xml** настраивает JPA:
- Источник данных: `java:/Web3DS`
- Провайдер ORM: EclipseLink
- Стратегия генерации схемы: UPDATE
- Диалект БД: PostgreSQL

**web.xml** настраивает сервлет:
- FacesServlet обрабатывает все файлы *.xhtml
- Стартовая страница: start.xhtml

## Запуск проекта

### Предварительные требования
- JDK 17 или выше
- Gradle (или использовать bundled gradlew)
- Docker и Docker Compose (для контейнеризованного запуска)
- PostgreSQL 15 (или использовать Docker)

### 1. Локальный запуск (через Gradle)

```bash
# Сборка проекта
gradlew.bat build

# Развёртывание WAR-файла
# Скопируйте build/libs/lab_3.4.war на сервер приложений
# Поддерживаемые серверы: Tomcat 11+, Payara 6+, WildFly 27+
```

После развёртывания приложение доступно по адресу:
```
http://localhost:8080/lab_3.4/
```

### 2. Запуск через Docker Compose

```bash
# Запуск всех сервисов (PostgreSQL + приложение)
docker-compose up --build

# Приложение будет доступно на http://localhost:8080/lab_3.4/
```

### 3. Быстрый перезапуск

Для удобства разработки предусмотрены скрипты:
- `direct_restart.bat` - быстрая перезагрузка на локальном сервере
- `docker_restart.bat` - перезагрузка Docker контейнеров
